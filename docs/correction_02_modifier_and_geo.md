## 0. 何が起きたか

第1回（186ページ・文字列駆動）は過剰分割、第2回（100ページ・SERP重複ページ）は逆に過剰統合。
振り子を止める安定解は **2信号：SERP + 修飾子次元**。

| 失敗 | 証拠（第2回実データ） |
|---|---|
| 過剰統合 | 「aga治療 オンライン おすすめ」= member 105 / cover 423、「治療薬 市販」70、「aga治療 安い」26 |
| 別意図の潰れ | おすすめの修飾子別意図（料金/オンライン/皮膚科/女性/年代/ランキング）が統合され消失 |
| 地域の軸散乱 | 地域が6軸に分裂（t_02/03/04/05/08/10）。「地域別おすすめクリニック」名前3軸。t_04はオンライン(非地域)混在 |
| 軸のgrab-bag | t_01「基礎知識」に 安い(費用)・効果いつから・おすすめ・皮膚科おすすめ・選び方 が雑多に吸収 |
| 地域階層未構築 | parent_location 57/100 のみ充填 |

---

## 1. 根本原因（1点）

軸割当とページ統合が **同じ単一信号（SERP/embeddingクラスタ）のflat適用**。
- 修飾子の別意図を弁別する次元が無い → 過剰統合
- 各都市SERPが局所的に異なる → 地域が多数クラスタ（多数軸に散乱）
- 軸が emergent label（クラスタの自動命名）→ 名称重複・grab-bag

---

## 2. 修正A：ページ分割に「修飾子意図」次元を追加（過剰統合の是正）

ページ統合を **2段** にする。SERP重複で粗くまとめ、修飾子意図で細かく割る。

```
軸内のページ確定ロジック:
  STEP1 粗クラスタ : SERP重複 ≥ N でまとめる
  STEP2 細分割     : 粗クラスタ内を「修飾子意図」で分割
        修飾子の型（controlled・閉集合）:
          price     費用・安さ・相場
          channel   オンライン / 来院
          provider  皮膚科 / 専門クリニック
          audience  女性 / 年代(10/20/40代) / 学生
          format    ランキング / 比較 / 口コミ評判
          general   無修飾の総合おすすめ
```

**おすすめページは1本でなく複数**（あなたの意図）。例: 総合おすすめ / 料金重視 / オンライン特化 / 皮膚科 / 女性向け は別ページ候補。

---

## 3. 修正A'：カニバリ・セーフガード（過剰分割への逆戻り防止）→再証

修飾子分割は無条件ではなく、第1回の過剰分割に戻らないよう、**分割は下記を満たす時のみ**。

```
修飾子で分割する条件（AND）:
  ① 修飾子が distinct な sub-demand を持つ（同じcontrolled集合に該当）
  ② 上位3 SERP が完全一致でない、または各ページの content angle が実質的に異なる
分割しない場合:
  修飾子KWは member(passage) として総合ページに吸収
```

判別: 「aga 安い」と「aga おすすめ」の各ページが同一content・上位3完全一致なら**統合**（自己カニバリ回避）。
content angleが料金表中心 vs 総合比較で割れるなら**分割**。SERP重複は粗クラスタ用、content angle/上位3が細分割の最終判別。

---

## 4. 修正B：軸を controlled taxonomy 化（散乱・命名・grab-bagの是正）

軸を bottom-up クラスタの自動命名でなく、**少数の top-down 意図ファミリー** として確定する。

- 軸はSERPクラスタの emergent label にしない（名称重複の原因）。
- 軸の導出: ページ群を意図ファミリーへ分類（汎用ロジック・ハードコード禁止・結果はconfig記録）。
- **地域は定義上1軸**（複数軸に割らない）。
- grab-bag禁止: 費用・効果・副作用・基礎知識・クリニック選びは別軸。t_01のような混在を解消。

AGAで想定される軸の形（叩き台・検証用・固定値ではない）:
```
クリニック選び・おすすめ / 地域 / 治療薬 / 効果・経過 / 費用 /
副作用・リスク / 基礎知識 / 予防・セルフケア / 対象識別 / 施術タイプ
```

## 5. 修正C：地域の構造化（あなたの#3を全て同じ軸・構造化して整理）

地域は単一軸とし、`parent_location` で階層化する。

```
軸: 地域（単一）
  parent_location 階層: 地方 > 都道府県 > 市区 > 駅・エリア
    例) 関東 > 東京都 > 新宿区 / 渋谷区 ...
        近畿 > 大阪府 > 梅田 / 難波 ...
  ページ生成ゲート（correction_01 §4を継承）:
    表記ゆれ統合 ∧ SERP独自性 ∧ ノイズ除去 ∧ 独自ローカルcontent可能
  roll-up:
    薄い都市（vol=0・SERP非独自）は独立ページにせず親(都道府県/地方)ページへ集約
    → doorwayリスク回避 ＋ 構造的整理を同時達成
```

- parent_location は全地域ページで充填（現状57/100→100%）。
- 「東京国際クルーズターミナル」等の極小ノイズは親へ吸収または除外。

## 6. 受け入れ基準（再実行後）

## 8. 振り子の収束（設計整合）

```
第1回: 文字列駆動 → 過剰分割（同義語が別ページ）
第2回: SERP重複のみ → 過剰統合（別意図が1ページ）
今回解: 軸=controlled taxonomy(top-down)
        ページ= SERP重複(粗) × 修飾子意図(細) × content-angleガード(カニバリ防止)
        地域= 単一軸 + parent_location階層 + roll-up
```

細かく(fine)：修飾子の別意図にも厳格に(strict)：同義語の統合判定にも。判別は SERP重複＋content angle＋修飾子型を両立させる。
