# 修正仕様 01：軸レイヤー・同一意図マージ・出力契約

最終更新: 2026年5月25日
位置づけ: `docs/requirements.md` の §3[L3] / §4 / §7 を上書き・補強する。本書が優先。
対象: AGAパイロット第1回出力（186 page / 2213 member行）の構造欠陥の是正。

---

## 0. 根本原因（1点）

クラスタリングと軸割当が **キーワード文字列パターン駆動** になっており、**検索意図（SERP）駆動** になっていない。
このため (a) 同一意図が別ページに散逸し、(b) 地域が過剰分裂し、(c) 軸（theme）が文字列prefixに誤って埋め込まれている。

修正の核: **軸割当・ページ統合の判定基準を、KW文字列ではなくSERP重複（意図のグラウンドトゥルース）に置く。**

## 1. 実データが示す欠陥（是正対象）

| 欠陥 | 証拠 |
|---|---|
| 同一意図の別ページ化 | 「aga おすすめクリニック」(core:) /「aga治療ランキング」(trust:) /「agaクリニック 評判」(trust:口コミ) /「AGA 皮膚科 おすすめ」(format:皮膚科) /「aga治療 安い」(cost:) が別bucket・別page。SERPはほぼ同一 |

## 2. 修正A：軸（theme）レイヤーの導入 ＝ §4/§L4 上書き

3層階層を明示する。

```
軸 theme（少数・SERP意図ファミリーで導出）
  └ page（軸内の真に別意図なクラスタ）
      └ member_kw（passage・UI専用）
```

- 軸は **KW文字列でなくSERP意図ファミリー** で導出する（同prefix=同軸、ではない）。
- 軸数は少数（AGAで概ね8〜12を想定。下記§6の受け入れ基準で検証）。
- 軸はハードコードしない。汎用的に導出し、結果をconfig/master tableに記録（Daiki承認後に凍結）。
- 現 `bucket` prefix は廃止または軸へ再マップ（文字列割当をやめる）。

---

## 3. 修正B：同一意図マージ ＝ §3[L3] 上書き（あなたの#1）

ページ統合の判定をSERP重複に一本化する。

```
判定ルール:
  page_i と page_j の上位N URL重複 ≥ しきい値  → 同一ページに統合（軸prefixを跨いでも統合）
  重複 < しきい値                              → 別ページ（軸が同じでも別page）
```

- 「おすすめ／ランキング／比較／評判／皮膚科おすすめ／安い（推奨文脈）」は、SERP重複が高ければ**1ページに統合**（現状は5+ページに分裂）。
- 文字列の同義（おすすめ＝ランキング＝評判）はembeddingでなくSERPで同一性を確定する。
- しきい値Nと重複率は §6校正ゲートでDaikiが決定。

## 4. 修正C：地域の扱い（あなたの#4の精緻化）

地域カバレッジは競合が手作業で取りこぼす領域＝差別化の堀。ただし**真に別SERPの都市だけ**をページ化する。

```
地域ページ生成ゲート（必然性テスト）:
  ① 同一商圏の表記ゆれは統合
     例: 東京/都内/東京都 → 1。新宿×3 → 1。大阪/梅田/西梅田/大阪梅田 → 商圏で統合
  ② SERP独自性: 当該都市SERPが親地域と有意に異なる場合のみ独立ページ
  ③ ノイズ除去: 駅・ターミナル等の極小粒度（例「東京国際クルーズターミナル」）は除外/吸収
  ④ 独自content可能性: その都市の実クリニック等、独自ローカル情報を書けるpage のみ生成
```

**⚠️ doorwayリスク（材料・要判断）**: 新規YMYL医療ドメインで、vol=0・member=1の薄い都市ページを大量生成すると、Googleの誘導ページ(doorway)/thin contentに該当しうる。各都市ページが**実ローカル情報を持つ**なら堀になるが、テンプレ量産は新規ドメインで逆効果になりうる。地域粒度の下限はDaiki判断（§6ゲート）。

## 5. 修正D：出力契約 ＝ §7 上書き（あなたの#2）

CSVをページ単位に簡素化。member_kw（passage）はCSVに出さず、UIで確認可能にする。

### 新CSV（1行=1ページ）
```
page_id, theme(軸), page_rep_kw, intent_layer, page_cover_size, pagerank, rep_volume, action
```
- `theme` = 軸（新規）
- `page_rep_kw` = 旧 axis_kw（ページ代表KW・名称明確化）
- member_kw 列・is_representative 列・個別KMのvol/kd/cpc/intentは **CSVから除去**
- 行数 ≈ ページ数（旧2213行 → 数十行規模）

### UI（kw.anonymous-seo.jp）
- ページ詳細で member_kw 全件（vol/kd/intent付き）を確認できる
- 軸→ページ→memberのドリルダウン
- カバレッジ（どのKWがどのページに吸収されたか）を可視化

## 6. 受け入れ基準（再実行後に検証）

| 項目 | 合格条件 |
|---|---|
| 軸数 | 少数（概ね8〜12）。文字列prefixでなくSERP意図で導出されている |
| おすすめ・比較クラスタ | core/ランキング/評判/皮膚科おすすめ/安い(推奨)が**1ページ**（または極少数）に統合 |
| 東京エリア | 23ページ → 商圏統合・ノイズ除去後の妥当数（新宿の3重複は1に） |
| 地域ページ | 全て§4ゲート通過（表記ゆれ統合済・ノイズ除去済・独自content可能） |
| CSV | 1行=1ページ。member列なし。行数=ページ数 |
| UI | member全件がページ単位で確認可能 |

## 8. #1と#4の関係（設計上の整合確認）

#1「同一意図は統合（ページ減）」と #4「細かく厳格に（カバレッジ増）」は矛盾しない。粒度の軸が違う。

```
細かく(fine)   → 真に別の意図に適用（都市ごと・治療薬種別ごと）＝競合が取りこぼす堀
厳格に(strict) → 「同一意図」の定義に適用（同義語はSERPで同一判定し統合）
```

判別器は **KW文字列でなくSERP重複**。同義語は統合し、真の別意図は分割する。これがカニバリ予防（統合）と専門性カバレッジ（分割）を同時に満たす。
